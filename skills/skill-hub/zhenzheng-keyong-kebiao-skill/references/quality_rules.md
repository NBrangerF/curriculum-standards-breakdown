# 质量规则

## 硬规则

- 不编造标准 code。
- 不编造课程标准原文。
- 不把“基于课标生成的建议”写成课程标准原文。
- 找不到标准时输出“未在当前数据版本中找到”，并说明下一步需要的数据。
- 数据版本缺失时明确写出“未找到独立数据版本文件”。
- 学段口径不统一时标注“不确定”，不要输出确定结论。
- 对上传文件的解析必须保留 `warnings` 和置信度。
- 课表必须做约束检查。

## 匹配分数解释

- `0.80-1.00`：高置信度，主题、目标和标准核心动词/对象高度一致。
- `0.55-0.79`：中置信度，存在关键词或领域一致，但需要人工确认。
- `0.30-0.54`：低置信度，只能作为候选，不建议直接采用。
- `<0.30`：不要作为正式匹配输出，除非用户要求看候选。

## 匹配字段

优先匹配：

1. `subject_slug`
2. `grade_band`
3. `domain`
4. `subdomain`
5. `standard`
6. `practice`
7. `teaching_tip`
8. `assessment_evidence_type`
9. `ts_primary` / `ts_secondary`

## 缺失数据处理

- 用户问“现行教学计划”但未提供计划文件：说明当前只能基于现有课程标准数据和用户条件查询，不能代表某地现行教学计划。
- 用户指定的学科不在当前数据覆盖：列出当前覆盖学科。
- 用户指定教材版本：如果当前数据没有教材版本映射，说明缺失，不要推断。
