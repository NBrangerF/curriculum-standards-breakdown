# 安全与质量

## 不编造

- 标准 code 必须来自数据库或 API。
- 标准正文必须原样来自数据库或 API。
- 不能用通用教育常识补齐缺失标准。

## 数据版本

- 每个结果都要显示数据版本。
- 没有 `data_version.json` 时，使用 manifest 生成时间并说明未独立校验。
- 当前数据覆盖范围之外的问题要明确拒绝确定化回答。

## 低置信度

- 上传文件解析的低置信度字段进入 warnings。
- 匹配分数低于阈值时标为“待人工确认”。
- 学段口径不统一时输出“需确认学段口径”。

## 原文与建议分离

输出中明确区分：

- 课程标准原文
- 匹配理由
- 基于课标生成的教学建议
- 基于课标生成的评价建议
