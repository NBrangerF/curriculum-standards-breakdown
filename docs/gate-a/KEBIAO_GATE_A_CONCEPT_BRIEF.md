# kebiao Gate A：视觉与交互概念说明

- 状态：待确认
- 对应主计划：`docs/KEBIAO_VISUAL_INTERACTION_GRAPH_MASTER_PLAN.md`
- 范围：视觉系统、现有页面的交互质感、知识图谱工作台概念
- 非范围：路由、信息区块、字段顺序、搜索规则、数据契约、导航顺序

## 1. 设计主张

**kebiao 是中国课程标准的结构化索引与智能引擎。**

它应呈现为一套精准、可探索、冷静的课程知识工作台：用户看到的是课程结构本身，而不是“教育政务页面”、纸质教材、传统中国风或泛 AI 营销模板。

核心视觉签名是 **课程坐标系**：

- 学段是横向轴；
- 领域是并行轨道；
- 标准编码是可定位锚点；
- 真实关系是可追溯索引线；
- 当前选择在所有页面有一致的空间定位。

它不是装饰性网格，也不生成虚构关系或“智能”效果。

## 2. 视觉锁定

### 色彩与材质

| 角色 | Token 候选 | 用途 |
| --- | --- | --- |
| 基底 | `#F7F8FA` | 冷白工作区，非米白/纸张色 |
| 一级文字 | `#15181E` | 标题、代码、主要数值 |
| 二级文字 | `#59616E` | 说明、元数据、低优先信息 |
| 结构线 | `#D8DDE6` | 轨道、分隔、未选状态 |
| 信号靛蓝 | `#3D5AFE` | 当前选择、主操作、定位信号 |
| 深石墨 | `#10141C` | 仅图谱画布和需要空间感的局部表面 |
| 成功 | `#167D55` | 成功/已保存，不抢占主题色 |
| 注意 | `#B96A0A` | 需复核/候选状态 |
| 错误 | `#C33945` | 明确失败或不可用状态 |

- 全站只有一个主强调色：信号靛蓝；学科色仅用于数据分类，不承担按钮主题。
- 页面是冷白与淡石墨结构线；不能变成暖灰、纸张、红金，也不能以蓝紫渐变充当“AI 感”。
- 材质来自精确的 1px 边界、轻微内阴影、低对比噪点和统一光源；不用泛滥卡片或玻璃态。

### 字体与数字

- UI：`Geist` 为拉丁、编码和控制文字；中文优先 `Noto Sans SC` 或系统苹方作为回退。
- 正文：17px / 1.72；长段落控制在可阅读的行宽。
- 标题：不使用衬线、不使用宋体式出版物语言；标准标题采用紧凑粗体与适度负字距。
- 编码、数量、筛选条件、图谱坐标：等宽数字与 `font-variant-numeric: tabular-nums`。
- 不使用 emoji 作为导航、区块标题或状态图标；以统一的 1.5px 定制几何图标替换。

### 几何与容器

- 外层页面栅格：1440px 设计宽度，24px 基线；内容工作区而非营销卡片墙。
- 通用半径：4px 控件、8px 可聚焦面板、12px 仅用于强调性移动端抽屉；避免全站大圆角。
- 详情页、搜索页、审核页保留其现有密度，采用“开放内容区 + 有目的的边界/轨道”，不把表格或画布套进大卡片。
- 图谱是画布，不是手工摆放圆形/菱形教学示意图。

## 3. 三个表面的视觉与交互语言

### A. 首页：结构首次显影

首页仍保留“品牌与价值说明 → 对比筛选 → 学科入口 → 可迁移技能 → 使用说明”。

- 首屏把原有标题、说明和三项既有任务放入冷白空间；背景以缓慢显影的课程坐标线表达层级，不使用暗色横幅与泡泡。
- 原有“开始筛选”“按学科浏览”“按可迁移能力浏览”文案与目标不变。主操作是靛蓝实心按钮；另外两项为克制的文字/描边操作。
- 对比筛选成为具有清晰进度与已选锚点的结构面板：学科、学段、技能仍按照现有字段和规则呈现；选中时仅发生定位条滑动、边界着色和计数更新。
- 学科与技能入口使用轨道式列表/区域而不是九张同质卡片；每项仍完整可点。

### B. 标准详情：研究阅读与关系定位

标准详情仍保留全部内容及顺序。

- 顶部由“大号编码锚点 + 标准标题 + 实际元数据 + 原收藏/复制/分享操作”构成；不新增帐户、全局搜索或历史入口。
- 将分类信息从松散信息条升级为可扫描的坐标轨：学科/领域/学段/年级/标准名称是同一张结构图上的字段，而不是彩色 badge 堆叠。
- 正文段落、教学提示、证据、资源维持阅读尺度；仅以标题锚线、折叠状态和段落进入动效强化层次。
- 新增的“定位到图谱”只作为 Gate B2 后、已有关系信息区域内的并行入口；默认不出现，绝不替代相关标准和技能列表。

### C. 知识图谱：可验证的关系工作台

图谱的图标化表达借鉴 os-taxonomy 的可缩放知识空间与参考图的“三栏工作台”，但产品视觉、数据和交互均为 kebiao 自有。

- **左侧 LayerRail：** 学科、领域、标准、能力的可见性和范围筛选；每项具数量与当前状态，永不隐藏唯一信息路径。
- **中央 Spatial Canvas：** 深石墨画布。学段是水平轴，领域是纵向轨道；初始视图优先显示当前节点的局部一跳关系。线条按真实关系类型使用可辨识的实线、虚线和方向，而不是用颜色暗示未经验证的因果关系。
- **右侧 Inspector：** 标题、代码、类型、来源、关系证据、相关标准/技能列表和现有详情链接。节点选择只更新 Inspector 与高亮，不触发布局重算。
- **底部 ContextBar：** 当前范围、可撤销的筛选、缩放级别、键盘提示和“以列表浏览关系”入口。
- **等价可访问性：** `GraphA11yController` 和虚拟化关系树与画布共同交付；键盘可在节点、关系、Inspector 与列表之间移动。拖拽/滚轮不是唯一操作。

## 4. 状态与微交互规范

| 交互 | 可见反馈 | 时长 | 无障碍退化 |
| --- | --- | ---: | --- |
| 选择筛选 | 锚点滑入、已选计数更新、结果区域保持稳定 | 180ms | 立即更新文本与 `aria-live` |
| 详情区块进入 | 标题锚线与内容由 4px 位移淡入 | 220ms | 无位移，仅淡入或立即显示 |
| 视图切换 | 共享坐标线缩放/重排，内容交叉淡入 | 240–320ms | 即时切换，焦点移动到新视图标题 |
| 图谱聚焦 | 相邻节点增强，非相邻节点降到 24% 对比度 | 180ms | 关系树同步展开与选中 |
| Inspector 更新 | 内容骨架 → 真实数据淡入；画布不重排 | 160ms | 文本即时更新，保持焦点 |
| 收藏/复制/提交 | 受控图标填充与简短成功提示 | 160ms | 文本状态和可读提示 |
| 错误/空态 | 保留已有原因与恢复路径，禁止用动画掩盖 | 0–180ms | 同样可读 |

- 动画仅限 `transform` 与 `opacity`，所有运动遵从 `prefers-reduced-motion`。
- 不使用滚动劫持、永久漂浮、连续粒子、自动 3D 相机或仅 hover 才显示的信息。
- 将 GSAP 限定在获得许可证决定后的非关键装饰性过渡；关键 UI 状态使用可测的 CSS/React 状态机。

## 5. Gate A 图像概念清单

以下概念均为预览，不作为生产界面截图或可复制文案。它们必须共同遵守“原有 IA 不变、冷白工作台、石墨图谱、课程坐标系”这一方向。

1. 首页桌面：现有五个内容区块的首屏与筛选开端。
2. 标准详情桌面：真实标准阅读、关系定位和已有操作。
3. 图谱工作台桌面：LayerRail / Spatial Canvas / Inspector / ContextBar。
4. 搜索与对比状态：草稿筛选与应用筛选分离的可见状态。
5. 移动端标准详情：现有字段和操作的单列层次。
6. H4G 审核：高密度、低动效的队列与证据对照工作区。

## 6. Gate A 通过条件

- 品牌为 `kebiao`，副标题固定为“**中国课程标准的结构化索引与智能引擎**”。
- 没有暖纸、朱砂、金色、古典中国风、传统衬线、泛 AI 渐变、emoji 或营销化信息重组。
- 首页、详情、搜索、移动和图谱都呈现为同一套系统。
- 图谱明确是“可验证的关系工作台”，有 Inspector 与等价列表，不是装饰性网络。
- 不新增、不删除、不调换当前页面区块或导航项目；所有新入口都是平行增强。
- 确认后才能进入 Foundation Gate（Token/字体/图标/状态基础）和图谱数据/引擎基准；在此之前不得添加生产依赖或改写 UI。
