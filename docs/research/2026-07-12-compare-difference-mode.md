# 多学段对比差异模式

## 问题

主计划要求对比视图支持差异模式，并规定列切换和差异模式保持行位置，差异必须同时使用标记、背景和文字，不能只依赖红绿。原实现已经按领域对齐多学段列，但没有差异开关或显式差异语义。

## 差异定义

当前数据没有跨学段标准的一一语义配对 ID，因此不伪造文本级 diff。差异模式使用可验证的结构指标：同一领域在各学段中的标准数量。当数量集合不唯一时标为“数量差异”，否则标为“结构一致”。

## 实现

- 仅在单一学科且选择至少两个学段时显示“突出差异”。
- 开启后显示差异领域总数和“行顺序保持不变”说明。
- 每个领域头显示“数量差异 · 3 / 7”或“结构一致”。
- 每列显示“第一学段 · 3 条”等明确文本。
- 差异领域使用靛蓝边缘、轻背景和图案标记；一致领域使用中性边缘。
- 不隐藏、不排序、不折叠任何领域或标准。
- 不使用红绿作为状态编码。
- Motion `AnimatePresence` 只控制摘要和标记的 opacity/transform；全站 MotionConfig 继续处理 reduced-motion。

## 结构修复

CompareView 原先在多学科条件分支提前 return，后续仍声明多个 `useMemo`。当用户在同一组件实例中从多学科切换到多学段时，存在 Hook 数量变化风险。本次把所有 hooks 移到条件 return 之前，确保模式切换保持固定 Hook 顺序。

## 外部项目决策

- 继续使用已安装的 [motiondivision/motion](https://github.com/motiondivision/motion)；没有新增动画 runtime。
- 未引入 TanStack Table：当前差异逻辑仍是现有对齐 CSS Grid 的状态增强，不需要新的表格模型。
- `critique-skills` 只适合评价 Agent Skill 包，不适用于 UI 审查，因此本轮未错误套用。

## 验证

- 新增差异态 axe，critical/serious 为 0。
- 新增专用视觉快照，完整视觉回归 13/13。
- 完整 E2E 50/50；Stylelint、全 workspace typecheck、生产构建和 audit 通过。
- 13 路由 × 5 视口共 65 份基线通过；10,730 个默认态交互目标低于 44px 为 0。
- 主包 138.07 KB gzip，低于 150 KB；图谱 chunks 与 lazyGraphLeak 门禁不变。
