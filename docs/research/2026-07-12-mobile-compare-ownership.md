# 移动对比内容所属上下文

## 问题

主计划要求移动端重新容器化后始终显示内容所属对象。对比网格在 768px 以下折叠成单列，但 StandardCard 的桌面学段 chip 同时会隐藏。用户滚动离开顶部列头后，只能看到标准编码和正文，无法判断卡片属于哪个学段；多学科模式同样可能失去学科上下文。

## 实现

- StandardCard 增加可选 `contextLabel`，默认不渲染任何额外内容。
- 多学段 CompareView 传入“第一学段 · 1–2年级”等学段上下文。
- 多学科 SubjectColumn 传入“数学 · 第二学段”等学科与学段组合。
- context label 在桌面隐藏，仅在 768px 以下显示，避免重复列头信息。
- 标签位于标准编码之后、操作按钮之前，保持原卡片正文和动作顺序。
- 采用扁平浅色表面、2px 学科色索引和 mono 小字，不增加新卡片或阴影层。

## Skill 与外部项目边界

- `minimalist-ui` 用于保持标签扁平、紧凑和信息优先；未套用暖色主题、Bento 或大面积装饰。
- 不需要新依赖，继续使用现有 CSS Modules 与 StandardCard 组件契约。
