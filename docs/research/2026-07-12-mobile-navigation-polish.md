# 移动导航交互质感硬化

日期：2026-07-12

## 问题

原移动导航虽然具备汉堡图标变形与基本键盘操作，但菜单通过条件渲染瞬时出现，并把页面内容向下推移；缺少遮罩、退出过渡、Escape 关闭和焦点返回。行为可用，但触感仍像基础响应式菜单。

## 实现

- 使用现有 [Motion](https://github.com/motiondivision/motion) `AnimatePresence` 与 `m`，没有新增动画运行时。
- 菜单改为固定浮层，不改变文档流和首页首屏位置。
- 遮罩与菜单只动画 `opacity`、`transform`，链接以 45ms 错峰进入。
- 背景提高到 99% 白色，避免首页大标题穿透造成视觉噪声；活动项使用小半径行块和轻内描边。
- Escape 关闭后焦点返回菜单按钮；点击遮罩或导航后关闭。
- 统一 `MotionConfig reducedMotion="user"` 继续负责系统级减弱动效。

## 证据

- 移动菜单完整任务流连续 5 轮通过。
- 390 × 844 实际截图人工检查了打开状态、遮罩层级、菜单密度和内容不位移。
- 信息架构、五个导航项、URL 与 `aria-current` 保持不变。
