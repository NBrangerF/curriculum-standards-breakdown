# 教学计划到课标匹配评估

更新时间：2026-07-10

本评估为 `POST /api/v1/matching/plan-to-standards` 的确定性基线。它不评估教学计划的文笔，也不评估生成式内容；只验证系统能否在正确学科和学段内，为每个教学单元返回可解释、可复核的真实课程标准。

## 评估资产

- 夹具：`packages/curriculum-core/test/fixtures/plan-matching-fixtures.json`
- 评分器：`scripts/evaluate-plan-matching.ts`
- 命令：`npm run eval:matching`
- 绑定数据版本：`2026.07.09`

评分器会先读取 repository 的 `data_version.json`。数据版本与夹具不一致时直接失败，避免新旧课标混用后仍然报告“通过”。

## 覆盖范围

当前基线包含十份真实教学活动语义改写的计划、十一条单元断言，覆盖：

- 科学：土壤比较、自然观察，以及一个开放主题的人工复核保护案例。
- 数学、语文、艺术、英语、信息科技、劳动、体育与健康、道德与法治各一份。
- H1、H2、H3、H4G7 四个学段。

每一条单元断言可以包含：

| 字段 | 作用 |
| --- | --- |
| `acceptable_codes` | 经人工确认的可接受课标 code；只计算这些单元的检索排序指标。 |
| `forbidden_top_codes` | 语义相近但不应排第一的 code，用于拦截明显误命中。 |
| `min_top_score` | 防止低确定性结果被当作正常匹配。 |
| `min_matched_fields` | 要求输出至少足够数量的命中字段，并同时要求 `rationale`。 |
| `expected_requires_human_review` | 验证高确定性计划可直接使用、模糊计划必须进入人工确认。 |

## 质量门槛

当前 deterministic baseline 的发布门槛为：

| 指标 | 门槛 | 含义 |
| --- | --- | --- |
| 元数据准确率 | 100% | 计划必须落在正确学科和学段。 |
| Top-1 准确率 | >= 90% | 金标排第一的比例。 |
| Recall@3 | 100% | 每个有金标的单元，前三条中必须出现一个可接受 code。 |
| MRR | >= 0.93 | 正确 code 越靠前得分越高。 |
| 可解释率 | 100% | 返回命中字段达到断言，且有中文匹配说明。 |
| 复核预期准确率 | 100% | 模糊计划不会伪装成高确定性结果。 |
| 禁配项第一名 | 0 条 | 不允许指定的误匹配排第一。 |

语文名著案例刻意要求 `CN-H4G7-BOOK-007`。当前数据中相邻名著标准高度相似，基线会把它记录为排序信号，而不是把相邻 code 全部放宽成金标。这样后续优化词项权重、标准去重或引入 Meilisearch/embedding reranker 时，指标才会真实反映进步。

## 使用方式

在以下变更前后都运行：

```bash
npm run eval:matching
```

必须执行评估的变更包括：

- `planning.ts` 的解析、筛选、打分或 review threshold。
- `public/data` 中的标准正文、领域、能力、实践或证据字段。
- 检索 adapter、词项归一化、Meilisearch 索引字段和未来的 reranker。

新增计划样本时，应先由人工根据教学目标和课标正文填写金标与禁配项，再运行评估；不要把当前 API 返回的第一条直接写成金标。模糊计划没有唯一课标时，不填 `acceptable_codes`，只断言分数、解释和 `requires_human_review`。

## 后续扩展

下一批应优先补充真实但已脱敏的学校教学计划，并标记人工双评、分歧原因、教材版本和实际采纳结果。达到足够样本量后，将本基线拆分为开发集、回归集和发布保留集，避免针对固定样本调参。
