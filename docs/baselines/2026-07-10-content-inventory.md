## 4. 核心行为回归清单

### 首页与搜索

1. 首页读取 URL 中的 `subjects`、`bands`、`skills` 并恢复可见选择。
2. 首页仅在符合既有比较规则时允许“查看对比结果”。
3. “开始筛选课程标准”必须平滑定位到 `#compare-filter` 并使首个控件可用。
4. `/search` 必须维持 `draftFilters` 与 `appliedFilters` 分离；修改草稿不能提前改变结果。
5. 应用筛选前必须执行原有规范化与有效性校验；重置回到应用状态。

### 标准、技能与收藏

1. 标准详情必须保留面包屑、分类数据、正文、教学信息、资源和相邻标准。
2. `FavoriteButton` 的添加、取消、清单选择及视觉选中态都必须可用。
3. 技能页的关联标准筛选不改变技能详情自身内容。
4. 复制链接、打印、反馈、浏览器前进/后退仍使用真实路径和真实状态，不能换成静态演示。

### 可访问性与响应式

1. 所有可点击元素保留键盘焦点、`aria-label` 和可见文本路径。
2. 不把关键操作仅放入 hover、拖拽或动画结束后。
3. 390px 宽度时主导航折叠；内容不得横向溢出，筛选与操作仍完整可达。
4. `prefers-reduced-motion` 下，视觉切换有无运动的等价完成态。

## 5. 已观察的现状（仅作升级起点）

- 当前品牌使用“课标罗盘”、指南针与 emoji；这是视觉负债，不代表可改动导航内容的授权。
- 当前 tokens 同时存在 Ocean/Orca 与蓝紫/青色两套覆盖变量；后续必须以单一设计 token 层收敛，但逐页面接入以防回归。
- 首页与详情已经包含真实任务和数据，不新增营销页、假指标或替代页面来改变其责任。
- 标准详情示例：`/standards/MA-D2-GE-003`；该页面是详情升级和未来“定位到图谱”入口的内容基线。
