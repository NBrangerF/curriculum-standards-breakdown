# kebiao AI MVP 完成度审计

日期：2026-07-15
范围：智能检索、纯文本课程对齐工作台、可信数据边界、API 契约、隐私、可观测性与发布门禁。

## 结论

当前版本已经形成可独立降级的 AI MVP：网站只接收自然语言，`gpt-5-mini` 提交带原句证据的结构化候选，`nlq-v2` 负责冲突裁决和最终约束。课程标准 code、证据、来源和匹配理由仍全部由当前数据版本上的确定性检索决定。模型不可用时，教师仍可完成检索、候选复核与覆盖分析。

本轮审计发现并关闭了四个发布级缺口：模型调用前缺少去标识化、AI 与普通 API 共用限流、覆盖分析信任客户端候选对象、仓库没有真实的部署后 smoke 工作流。

## 明确延期，不阻塞当前 MVP

- embedding/向量排序：需要教师标注集证明相对 baseline 有显著收益后再启用。
- DOCX/PDF：需先完成恶意文档防护、临时存储、删除 SLA、PII 声明和失败恢复。
- 账户、团队协作与审核历史：需先定义权限、保留、导出和删除模型。
- Recall@K、MRR、nDCG：必须基于教师标注的版本化相关性集合，当前契约评测不报告虚假“准确率”。

## 发布判定

只有以下证据同时为绿才可发布：类型检查、数据/API 测试、智能检索与匹配评测、样式检查、前端构建、AI 工作流 E2E、全量 E2E、A11y、生产 API smoke。密钥仅存在部署平台 Secret；仓库和浏览器产物不得出现密钥。
